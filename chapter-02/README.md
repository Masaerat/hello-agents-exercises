# 第二章课后习题：智能体发展史

## 习题 2.1：物理符号系统假说与符号主义智能体

物理符号系统假说是符号主义时代的重要理论基础。请分析：

- 该假说的“充分性论断”和“必要性论断”分别是什么意思？
- 结合本章内容，说明符号主义智能体在实践中遇到的哪些问题对该假说的“充分性”提出了挑战。
- 大语言模型驱动的智能体是否符合物理符号系统假说？

### 参考答案

物理符号系统假说认为，智能可以通过对符号结构的操作来实现。它包含两个方向的判断：充分性论断是指，只要一个系统具备合适的符号表示和符号操作机制，就足以产生一般智能；必要性论断是指，任何能够表现出一般智能的系统，本质上都必须是某种物理符号系统。

符号主义智能体在实践中遇到的问题，主要挑战的是“充分性”。早期系统可以在规则明确、领域封闭的问题中表现良好，但一旦进入开放、模糊、动态的真实世界，就会暴露出知识获取瓶颈、脆弱性和常识不足等问题。专家系统需要人类专家手工写入大量规则，维护成本高；规则覆盖不到的新情况会导致系统失效；符号规则很难完整表示现实世界中大量隐含、模糊、经验性的知识。这说明“符号 + 规则”虽然能实现某些智能行为，但不足以轻松扩展到通用智能。

大语言模型驱动的智能体与物理符号系统假说的关系比较复杂。从内部机制看，大语言模型主要依赖神经网络中的分布式表示，并不是传统意义上显式、可读、可手工维护的符号系统。但从外部行为看，LLM 智能体会处理自然语言、生成计划、调用工具、组织中间步骤，这些过程又表现出一定的符号操作特征。因此，更准确的说法是：LLM 智能体不完全符合传统符号主义的物理符号系统假说，但它在行为层面重新引入了符号化的语言、计划和工具调用，是神经方法与符号方法结合的一种现代形态。

## 习题 2.2：MYCIN 与高风险领域专家系统

专家系统 MYCIN 在医疗诊断领域取得了显著成功，但最终并未大规模应用于临床实践。请思考：

- 除了本章提到的“知识获取瓶颈”和“脆弱性”，还有哪些因素可能阻碍专家系统在医疗等高风险领域的应用？
- 如果让现在的你设计一个医疗诊断智能体，你会如何设计系统来克服 MYCIN 的局限？
- 在哪些垂直领域中，基于规则的专家系统至今仍然可能比深度学习更合适？请举例说明。

### 参考答案

MYCIN 没有大规模进入临床，不只是因为知识获取瓶颈和系统脆弱性，还涉及伦理、法律、责任归属、用户接受度和工程集成等问题。医疗诊断属于高风险场景，错误建议可能直接影响患者生命健康。如果系统给出错误诊断，责任应由医生、医院、系统开发者还是知识库维护者承担，并不清晰。医生也未必愿意信任一个无法完全解释其边界和风险的系统。除此之外，专家系统需要与医院信息系统、病历、检验流程和临床工作流整合，这在当时也很困难。

如果今天设计医疗诊断智能体，应该避免让系统独立替代医生，而应把它定位为辅助诊断和决策支持工具。系统可以包含结构化病历读取、医学知识库检索、指南匹配、鉴别诊断生成、风险提示、证据引用和人类医生确认等模块。对关键结论应提供来源和置信度，不能凭空生成；对高风险建议应强制人工审核；对患者隐私要进行严格权限控制和日志记录；对系统表现要持续评估，包括误诊、漏诊、偏见、鲁棒性和医生反馈。

基于规则的专家系统在一些边界清晰、规则稳定、可审计性要求高的领域仍然很有价值。例如税务合规审核、保险理赔规则校验、银行反洗钱中的硬性规则、工业设备安全联锁、药物相互作用禁忌检查等。这些场景的目标不是开放式创造，而是稳定执行明确规则。相比深度学习，规则系统更透明、更容易验证，也更适合满足合规审计要求。

## 习题 2.3：扩展 ELIZA 聊天机器人

在 2.2 节中，我们实现了一个简化版 ELIZA 聊天机器人。请在此基础上进行扩展实践：

- 为 ELIZA 添加 3-5 条新的规则，使其能够处理更多样化的对话场景，例如工作、学习、爱好等。
- 实现一个简单的“上下文记忆”功能，让 ELIZA 能够记住用户在对话中提到的关键信息，例如姓名、年龄、职业，并在后续对话中引用。
- 对比扩展后的 ELIZA 与 ChatGPT，列举至少 3 个维度上的本质差异。
- 为什么基于规则的方法在处理开放域对话时会遇到“组合爆炸”问题，并且难以扩展维护？能否用数学方式说明？

### 参考答案

扩展 ELIZA 的核心思路仍然是模式匹配和模板替换。可以添加类似规则：

```python
rules = [
    (r".*我叫(.+)", "很高兴认识你，{name}。你今天想聊些什么？"),
    (r".*我在(.+)工作.*", "你在{job}工作听起来很有意思。这份工作最让你有压力的地方是什么？"),
    (r".*我喜欢(.+)", "你喜欢{hobby}多久了？它通常会带给你什么感受？"),
    (r".*我正在学习(.+)", "学习{subject}时，你觉得最困难的部分是什么？"),
]
```

上下文记忆可以用一个简单字典实现，例如 `memory = {"name": None, "job": None, "hobby": None}`。当用户说“我叫小李”时，系统用正则提取“小李”并写入 `memory["name"]`；当后续用户说“我最近很累”时，系统可以回复：“小李，你最近很累，是工作上的事情比较多吗？”这类记忆并不代表真正理解，它只是把匹配到的字段保存并在模板中复用。

扩展后的 ELIZA 与 ChatGPT 至少有三点本质差异。第一，ELIZA 依赖人工规则和模式匹配，ChatGPT 依赖大规模数据预训练得到的语言模型能力。第二，ELIZA 只能处理预先覆盖的表达方式，ChatGPT 能泛化到大量未见过的语言形式和任务。第三，ELIZA 没有真正的语义理解和推理能力，ChatGPT 可以基于上下文进行总结、推理、改写和工具调用。第四，ELIZA 的知识来自规则编写者，ChatGPT 的知识主要来自训练数据和外部工具。

组合爆炸可以这样理解：假设开放域对话中有 `n` 类主题，每类主题有 `m` 种常见表达，每种表达又可能有 `k` 种上下文状态，那么规则数量可能随 `n * m * k` 增长。如果还要处理多个主题组合、用户省略、指代、反问和跨轮记忆，规则数量会进一步接近指数式增长。人工维护这些规则会迅速变得不可行，而且不同规则之间还可能互相冲突。

## 习题 2.4：心智社会与多智能体系统

马文·明斯基在“心智社会”理论中提出：智能源于大量简单智能体的协作，而非单一的完美系统。请分析：

- 在图 2.6“搭建积木塔”的例子中，如果 `GRASP` 智能体突然失效，整个系统会发生什么？这种去中心化架构的优势和劣势是什么？
- 将“心智社会”理论与现代多智能体系统（如 CAMEL-Workforce、MetaGPT、CrewAI）进行对比，它们之间有哪些关联和差异？
- 现在的大语言模型和智能体往往拥有强大的推理能力，这是否意味着“心智社会”理论在大语言模型时代不再适用？

### 参考答案

如果 `GRASP` 智能体失效，搭建积木塔系统会失去抓取积木的能力。即使其他智能体仍然能识别目标、规划顺序或判断塔是否稳定，系统也无法完成实际操作。这说明多智能体协作系统中，局部模块失效可能影响整体任务，尤其是当该模块处在关键行动链路上时。

去中心化架构的优势是模块职责清晰、可组合、可替换，复杂智能可以由许多简单能力协同产生。系统不必依赖一个全知全能的中央模块，不同子智能体可以分别处理感知、规划、抓取、检查、修正等任务。劣势是协调成本更高，模块间接口和通信协议容易成为瓶颈；如果缺少全局一致性控制，系统可能出现冲突、重复工作或责任不清。

现代多智能体系统与心智社会有明显关联：它们都把复杂任务拆解给多个具有不同角色或能力的智能体，通过协作完成目标。例如 MetaGPT 会模拟产品经理、架构师、工程师等角色，CrewAI 会让多个 agent 按职责分工执行任务，CAMEL 强调角色扮演式协作。这些都体现了“智能来自协作结构”的思想。

差异在于，心智社会中的许多智能体可以是非常简单甚至“无心”的过程，而现代多智能体系统中的每个 agent 往往由大语言模型驱动，单个 agent 本身就具备较强推理和语言能力。现代系统还更强调提示词、工具调用、工作流编排、共享记忆和执行环境。

大语言模型时代并没有让心智社会理论失效。相反，它让该理论有了新的工程实现方式。即使单个 LLM 很强，复杂任务仍然需要分工、审查、工具执行、记忆管理和安全控制。多个简单或复杂 agent 的协作，仍然是构建可靠智能系统的重要方法。

## 习题 2.5：强化学习、监督学习与智能体

强化学习与监督学习是两种不同的学习范式。请分析：

- 用 AlphaGo 的例子说明强化学习的“试错学习”机制是如何工作的。
- 为什么强化学习特别适合序贯决策问题？它与监督学习在数据需求上有什么本质区别？
- 如果要训练一个会玩超级马里奥游戏的智能体，分别使用监督学习和强化学习需要什么数据？哪种方法更合适？
- 在大语言模型训练过程中，强化学习起到了什么关键作用？

### 参考答案

AlphaGo 的强化学习机制可以理解为：智能体在棋局环境中选择落子，环境返回新的棋盘状态，最终胜负或局面评估形成奖励信号。通过大量自我对弈，系统不断试错，强化更可能带来胜利的策略，削弱效果不好的策略。它不是只模仿人类棋谱，而是在与环境交互中发现和改进策略。

强化学习特别适合序贯决策问题，因为当前行动会影响未来状态和未来收益。围棋、游戏、机器人控制、自动驾驶等任务都不是单步分类问题，而是需要在一连串行动中最大化长期回报。监督学习通常需要输入和正确标签，例如“这个局面应该下在哪里”；强化学习则需要环境、行动空间和奖励信号，智能体可以通过探索收集经验。监督学习学的是从样本到标签的映射，强化学习学的是在状态中选择行动以获得长期收益的策略。

训练超级马里奥智能体时，如果使用监督学习，需要大量人类玩家的游戏画面、操作记录和对应动作标签，模型学习“看到这个画面时人类按什么键”。这种方法可以模仿玩家，但可能难以超过数据中的行为，也不容易处理没见过的关卡状态。如果使用强化学习，需要游戏环境、可执行动作、状态观察和奖励设计，例如前进距离、得分、存活、通关奖励、死亡惩罚等。对这个任务来说，强化学习更合适，因为游戏本身是典型的序贯决策环境，智能体可以通过反复试错学习长期策略。

在大语言模型训练中，强化学习常用于让模型输出更符合人类偏好和任务目标。典型流程是先进行预训练，再通过指令微调让模型学会遵循任务，之后使用人类反馈或 AI 反馈训练奖励模型，并用强化学习方法优化模型输出。这类方法帮助模型减少有害回答、提升有用性、遵循指令，并在多轮任务中更贴近人类期望。

## 习题 2.6：预训练-微调范式与知识获取

预训练-微调范式是现代人工智能领域的重要突破。请深入思考：

- 为什么说预训练解决了符号主义时代的“知识获取瓶颈”问题？它们在知识表示方式上有什么本质区别？
- 预训练模型的知识绝大部分来自互联网数据，这可能带来哪些问题？如何缓解这些问题？
- 你认为“预训练-微调”范式是否可能被某种新范式取代？或者它会长期存在？

### 参考答案

符号主义时代的知识获取依赖人类专家把知识显式写成规则、事实和逻辑关系，这个过程成本高、速度慢、覆盖范围有限，而且很难表达常识、经验和模糊语义。预训练通过在大规模语料上自监督学习，把大量语言模式、世界知识和任务模式压缩进模型参数中，从而绕开了完全手工编码知识的瓶颈。

二者的知识表示方式有本质区别。符号主义知识通常是显式的、离散的、可读的规则或符号结构，例如“如果发热且白细胞升高，则考虑感染”。预训练模型的知识则是隐式的、分布式的、统计性的，存储在神经网络参数中。它不容易被直接阅读和编辑，但具有更强的泛化能力和语言适应性。

互联网数据也会带来问题。第一，数据可能包含错误、过时、偏见、歧视、低质量内容和版权风险。第二，模型可能记住隐私信息或敏感内容。第三，互联网语料分布不等于真实世界需求分布，模型可能在专业场景中不可靠。缓解方法包括高质量数据筛选、去重、隐私清洗、版权治理、偏见评估、领域微调、检索增强生成、事实校验、人工反馈和安全对齐。

预训练-微调范式很可能会长期存在，但形态会演进。未来可能出现更强的在线学习、持续学习、世界模型、具身智能、多模态交互、工具增强和自我改进机制，但这些新范式未必完全取代预训练，而可能与预训练结合。预训练提供通用基础能力，微调、检索、工具使用和强化学习负责把能力适配到具体任务和环境。

## 习题 2.7：从三个时代设计代码审查助手

假设你要设计一个“智能代码审查助手”，它能够自动审查代码提交（Pull Request），概括代码的实现逻辑、检查代码质量、发现潜在 bug、提出改进建议。请分析：

- 如果在符号主义时代（1980 年代）设计这个系统，你会如何实现？会遇到什么困难？
- 如果在没有大语言模型的深度学习时代（2015 年左右），你会如何实现？
- 在当前大语言模型和智能体时代，你会如何设计这个智能体的架构？它应该包含哪些模块？
- 对比三个时代的方案，说明智能体技术的演进如何使这个任务从“几乎不可能”变为“可行”。

### 参考答案

在符号主义时代，可以用静态规则和专家系统来实现代码审查助手。系统会把代码解析成语法树，然后用人工编写的规则检查命名规范、空指针风险、资源释放、圈复杂度、禁止 API、简单安全漏洞等问题。困难在于规则覆盖范围有限，难以理解业务意图和跨文件上下文，也很难概括实现逻辑。每种语言、框架和团队规范都需要大量手工规则，维护成本很高。

在 2015 年左右的深度学习时代，可以训练模型识别代码缺陷模式或进行代码分类。系统可以使用代码 token、抽象语法树、控制流图、历史 bug 数据和提交记录训练缺陷预测模型，也可以结合静态分析工具提高覆盖率。这比纯规则方法更能发现统计模式，但仍然需要大量标注数据，解释性有限，对复杂语义、需求背景和自然语言评论的处理能力不足，也很难像人类 reviewer 一样给出清晰修改建议。

在当前大语言模型和智能体时代，可以设计一个由多个模块组成的代码审查智能体：代码变更解析模块读取 diff、文件结构和依赖关系；上下文检索模块查找相关代码、测试、文档和历史提交；LLM 推理模块总结改动意图、识别风险并生成建议；工具执行模块运行测试、lint、类型检查和静态分析；安全规则模块检查敏感信息、权限、注入风险和依赖风险；审查报告模块输出按严重程度排序的问题、证据位置和修改建议；人工反馈模块允许开发者确认、忽略或修正建议。

三个时代的差异体现了智能体技术的演进。符号主义时代能处理明确规则，但难以理解上下文；深度学习时代能从数据中学习缺陷模式，但仍缺乏可靠推理和解释；大语言模型时代具备语言理解、代码理解、工具调用、上下文整合和报告生成能力，使自动代码审查从单点检查工具变成接近协作者的智能体系统。它仍然不能完全替代人类 reviewer，但已经可以在许多场景中提供实际价值。
